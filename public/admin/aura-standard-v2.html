§1 · Color

Color is law, not choice

اللون قانون لا اختيار: لوحة مرجعية في الأسفل، وأسماء دلالية في الأعلى، ولا قيمة لونية خام داخل أي مكوّن — أبداً

1.1
Tokens only. A raw hex or fixed palette class on a user surface is a build failure, not a style choice.القيمة اللونية الخام داخل أي مكوّن خطأ بنائي يُرفض، لا ذوق يُناقش.
1.2
Two layers, one grammar. Reference palette beneath (bronze-500, green-700 — never used directly); semantic tokens above (role-property-variant) — the only names components speak. New tokens require a need no existing token serves, an alias into the palette, and both mode values at birth.
1.3
The mark test. If removing it changes the meaning, it's text or icon — contrast and label apply. If not, it's decoration — brand bronze, exempt, silent (§3).
§2 · Typography & Space

Four voices, a numbered ramp, eight steps of air

أربعة خطوط لا خامس لها، وسلّم مقاسات مُسمّى، وثمانية مقادير للمسافة — و«تقريباً صحيح» تقاعدت
RAMPEN (DM SANS / CORMORANT)AR (CAIRO — +1px, taller leading)DUTY

body15 / 1.65١٦ / ١٫٩Reading text, UI copy

2.1
Duties are exclusive. Cormorant: display only, never UI controls, never Arabic. DM Sans: body and interface. Cairo: all Arabic, 500/600, never italic, never letter-spaced. JetBrains Mono: data only, never sentences. A fifth font is a bug.
2.2
Space steps: 4 · 8 · 12 · 16 · 24 · 32 · 48 · 64. Off-step gaps fail review. Touch targets ≥44pt.
2.3
Cairo parity. Bilingual surfaces pair sizes from the ramp's Arabic column so both scripts sit optically even.
2.4
Font delivery is design. The four families preloaded, font-display: swap, Arabic subset for Cairo — the brand arrives with the page, on every connection.
§3 · Icons & Marks

One icon library. Four sacred marks. Two of them have direction.

مكتبة أيقونات واحدة وأربع علامات: ما يحمل معنى يلتزم بالتباين وينطق، وما يزيّن يلبس البرونز ويصمت
3.1
Lucide only. Stroke 1.6 · sizes 16/20/24 · ink or muted, never bronze. No second set, no filled variants, no emojis as structural icons.
3.2
The mirror list. Flips in RTL: arrows, chevrons, send, undo/redo, list-indent. Never flips: clock, search, download, person, settings — anything physical or symmetric. One CSS rule keyed to dir.
3.3
Marks. ✦ ◆ · are symmetric and never change. ↳ is directional: Arabic content uses ↲ (U+21B2). Marks are typography in --bronze, used sparsely: ✦ ceremony and primary voice; ◆ Arabic list points and keep-outs; · joins «phrases».↳ للإنجليزية و↲ للعربية — السهم يتبع لغة جملته.
3.4
Marks are silent; icons speak. In Aura's UI every mark carries aria-hidden="true"; meaningful icons carry aria-label in the surface's language. (Marks inside exported posts are punctuation — §12.)
3.5
In Arabic posts only: 📍⚠️✅❌ as section markers, max 2–3 per post, never in the hook or the closing question.
§4 · Numbers, Dates & Register

Digits follow their sentence; Arabic follows the Gulf

الرقم يتبع لغة جملته، والعربية هنا خليجية مهنية معاصرة — يونيو لا حزيران، و«٪» قبل الرقم
4.1
The sentence rule. Standalone data — scores, counters, ratios, IDs, chip timestamps — is Western + Mono in every language: SCORE 77 · 3/3. A number inside running Arabic text is Arabic-Indic in Cairo: «منذ ٣ أيام». One question decides: does it sit inside Arabic prose?
4.2
Number agreement. 1 = singular «يوم واحد» · 2 = dual, no numeral «يومين، منشوران» · 3–10 = plural «٣ أيام» · 11+ = singular accusative «١١ يوماً». Every templated count string follows the table. «منذ ٢ أيام» is machine Arabic and forbidden.
4.3
Gulf register. Contemporary Gulf professional Arabic. Months: يناير، فبراير… يونيو، أكتوبر — never حزيران، تشرين. Percentages inside Arabic prose: ٨٢٪. Time displays Asia/Riyadh by default.

Components

§5 · The Five Primitives & Their States

Built once, bypassed never — and a component is five states, not one picture

خمسة مكوّنات أساسية لا تُتجاوز، ولكل مكوّن خمس حالات مكتملة: عادي، تحويم، تركيز، ضغط، معطّل
5.1
SectionHeader — 11px tracked uppercase in --bronze-text, hairline below. Arabic eyebrows: Cairo 600, untracked, no uppercase concept.
5.2
AuraCard — hairline border, 14px+ radius, one idea per card. No decorative shadows; depth is earned by hierarchy.
5.3
AuraButton — one bronze primary per view, verbs not nouns. States: hover deepens 8% · focus 2px --bronze-text ring offset 2px · active scale .98 · disabled 40% opacity, no pointer. The same state grammar applies to cards, chips, and rows.
5.4
CollapsibleList + InfoTooltip — tooltips answer from the guide corpus, one source of truth. Instructions never live in placeholders.
5.5
A sixth primitive changes the Standard first — the surface doesn't improvise.
§6 · Chips & Badges

One chip anatomy; color never speaks alone

شكل واحد لكل وسم: إطار شعري، نص ١٢، نقطة دلالية اختيارية — واللون لا يتكلم وحده أبداً
Live Evergreen Emerging Faded العربية ✦English DraftShipped
6.1
Anatomy: hairline border, 999px radius, 12px label, optional leading dot carrying the semantic color. Dot always paired with the label — never color alone.
6.2
Max three per row, then "+N". The ✦ travels with the primary. Confidence stored 0–1, displayed ×100 — "82", never "0.82".
§7 · Motion, Layout & Waiting

Calm that responds — and thinking that narrates

حركة تؤكد ولا تستعرض، وتفكير يروي ما يفعله — الزر لا يتجمد أبداً، والانتظار يصبح عرضاً للعمل
7.1
Motion: 150–300ms native easing · tap feedback ≤120ms · one staggered reveal per page load, then stillness · prefers-reduced-motion honored.
7.2
Layout: hairlines .5px · radius 14px+/999px · ONE visualization per page · breakpoints ≤480 / ≤768 / desktop · columns stack in reading order · the view's primary action lives in the thumb zone on mobile.
7.3
The Waiting Law. Any operation >2s narrates honest stages in the user's language («كتابة المسودة بأسلوبك…»); >10s shows true progress; the trigger disables on first tap — no duplicates. Loading uses parchment skeletons, never bare spinners.
§8 · Forms

One recipe, both directions

وصفة واحدة لكل نموذج: التسمية فوق، فالحقل، فالمساعدة، فالخطأ الذي يسمّي الحل — والحقول اللاتينية جزر LTR
8.1
Anatomy: label above (start-aligned) → input → helper below → error that names the fix («الرابط لا يحتوي linkedin.com — تأكد من نسخه كاملاً»). Placeholders never carry instructions.
8.2
LTR islands. URL, email, phone and code fields carry dir="ltr" always — an island with an Arabic shore. Label and helper remain RTL.
8.3
Errors use --error and §11's no-blame grammar. Validation on blur or submit, never per keystroke.
§9 · Overlays, Undo & Confirm

Stop interrogating users about reversible actions

ما يمكن التراجع عنه يحدث فوراً مع «تراجع» — وما لا يمكن، يسمّي الشيء وعاقبته، والإلغاء هو الافتراضي
9.1
Layers: scrim = ink @40% · z-ladder: scrim 40, sheet 50, dialog 60, toast 70 · toasts anchor bottom-end and mirror with direction.
9.2
Undo over confirm. Reversible → instant + undo toast (6s): «تمت الأرشفة · تراجع». Irreversible → a confirm that names the object and the real consequence: «حذف ‹تقرير ديلويت›؟ ستفقد إشارتان أحد روافدهما.» Cancel holds focus. Destructive red appears only here.
PART III

Language

اللغة — العربية وُلدت هنا، والكلمة جزء من الواجهة، والمحتوى لا يخترع رقماً
§10 · Bidirectional by Birth

Arabic is a first language here

ليست ترجمة لاحقة: كل شاشة تنعكس وتتنفس وتقتبس صحيحاً بحكم البناء، والاتجاهان يلتقيان دون أن يتصادما
10.1
Mechanics: dir+lang set together (Cairo arrives by itself) · logical properties only (paddingInlineStart, never left/right) · SVG: textAnchor stays "start", the x flips.
10.2
Punctuation: Arabic strings end with Arabic marks — ؟ ، ؛ — never Latin. Embedded English terms sit in bidi isolation (<bdi>). English terms stay English inside Arabic: AI, dashboard, KPI.
10.3
Typesetting: no italics in Arabic — emphasis is weight 600 · no kashida justification — Arabic text aligns start, never justify · quotes are «هكذا», phrases joined by interpuncts ( · ).
10.4
Both at once: two sanctioned layouts — language tabs (the voice card pattern, ✦ on the primary) or side-by-side cards. Each block carries its own dir/lang, so mixing is structurally safe.
§11 · Voice & Microcopy

The interface speaks like a person of standing — to everyone

الواجهة تتكلم بكرامة: تسمّي الواقع ولا تتّهم، وتترك باباً مفتوحاً — وبصيغة محايدة لا تفترض جنس من تخاطب
11.1
Empty states invite: «بصمة صوتك لم تتشكل بعد. منشوراتك تعلّم أورا — وبصمتك تتشكل مع كل منشور.» Name what isn't yet + one concrete step. Bureaucratic MSA banned.
11.2
Errors blame the system: "That didn't go through — your draft is safe. Try once more." The user is never the subject of a failure sentence.
11.3
Success is proof, not confetti: «صُقلت البصمة من ٩ منشورات.» The number is the celebration; ceremony belongs to §14's moments.
11.4
Gender-neutral by default. CTAs and instructions use masdar nouns and nominal sentences — «صقل البصمة», «تعليم أورا» — addressing no one and welcoming everyone. Fully gendered address is permitted only when diagnostic_profiles carries gender. Suffixes addressing the object, not the person («عدّلها لتشبهك»), survive.
11.5
The Valley voice: declines in ink (never red), plainly stated, the smallest way back in the same breath: «إيقاعك هدأ هذا الأسبوع — منشور واحد يعيده ✦». No guilt, no FOMO, no dying streaks.
11.6
Hooks never imply the reader is failing. The reframe pattern: «أنت لا تعاني من X، أنت فقط لم…»
§12 · The Content Surface

What Aura writes under your name

كل رقم له أب من مقتطفاتك، والمنشور ينجو من رحلة اللصق، والوسمان من القائمة المعتمدة في السطر الأخير
12.1
The Integrity Law. Every figure, quote and named source in generated content traces to a captured fragment, shown as a provenance chip in the draft view. No fragment → no number; the post asks instead of asserts. Generation EFs are forbidden from improvising statistics.
12.2
Post anatomy (locked): hook without emoji → tension → one specific number mid-post → uncomfortable closing question → signature line → two hashtags.
12.3
Hashtags: exactly two, last line only, from the canon — #التحول_الرقمي #قطاع_المياه #قطاع_المرافق #حوكمة_التحول #تحقيق_القيمة #إدارة_التغيير #رؤية_2030 + English twins — in the post's own language. #mmahafzah reserved as the personal mark.
12.4
The Paste Test. Copy outputs plain text: \n preserved, ◆ ↲ as real characters, zero markdown. A post's meaning must survive with its marks stripped — marks decorate structure, never replace words.
12.5
Ask Aura's grounding contract: the signal cited in bold · why this week · a required next step · Chief-of-Staff register, both languages. Generic encouragement is a contract violation.

§14 · Peaks, Valleys & Moments

Emotion is a budget — spent at five peaks, six milestones, and the Gulf calendar

خمس قمم تنال الاحتفاء، وستة إنجازات فقط تقاطع، والتقويم الخليجي حاضر — وما بينها هدوء فندقٍ باهظ
14.1
The Five Peaks: first signal · Monday's "your week, ready" · the moment of shipping · a milestone falls · the Report leaves the building. These get the ✦, the ceremony, the crafted Arabic. Everywhere else stays deliberately quiet.
14.2
The Milestone Canon (exhaustive): first signal detected · first post shipped · first signal reaches Live · primary voice forms or changes · score crosses a tier boundary · first Report exported. user_milestones accepts no id outside the canon without an amendment.
14.3
The Valley Law governs every decline — §11.5's voice, never red, never a goad.
14.4
The cultural calendar: Ramadan (cadence shifts post-iftar), the two Eids (greeting + pause: «عيدكم مبارك — الأسبوع جاهز متى ما جهزت»), Saudi National Day Sep 23, UAE Union Day Dec 2. Aura knows what day it is, in the Hijri sense.
14.5
The Interruption Budget — three lanes: Transactional (user-caused: auth, invites) — always allowed, unbudgeted. Rhythm — one scheduled email, Monday. Moments — canon milestones + calendar, immediate but rare; Eid greetings are in-app by default. Never: guilt, FOMO, "we miss you". The caps are law, not settings.
§15 · Two Modes & Fresh Versions
After sunset · بعد الغروب

Night is derived, never designed — and yesterday's build never haunts today

15.1
Light canonical. Every surface designed, reviewed, demoed in light. Night exists because tokens exist — verified on every changed surface, never hand-crafted. After the token purge, a mode switch is a variable swap: discrepancy impossible by construction.النهاري هو الأصل، والليلي مشتق يُتحقق منه — والتبديل بينهما تبديل متغيرات لا إعادة تصميم.
15.2
The Fresh Version Law. The PWA detects a new build and offers a quiet refresh — «نسخة أحدث جاهزة · تحديث» — so no user lives in a cached yesterday, seeing bugs already fixed.
15.3
The tripwire stands: one more dark-mode legibility bug in production → the toggle hides, light-only launch, dark returns after.
§16 · Trust Surfaces

Trust is said at the moment of exposure

الثقة تُقال لحظة الانكشاف لا في سياسة مدفونة: سطر هادئ عند الالتقاط والتعليم والتصدير، وحقوق البيانات في متناول اليد
16.1
The three exposure lines. Capture, teach, and report export each carry one quiet lock-line: «منشوراتك لك وحدك — تُستخدم لصقل صوتك فقط، ولا تُشارك مع أحد.»
16.2
Data rights findable: export-my-data and delete-everything live in Settings, working, one tap deep.
16.3
The subprocessor list tells the whole truth. Every provider that touches user data is named in the privacy policy before it ships — an unlisted provider in production is a Standard violation. PDPL-aligned by design.
§17 · The Living Product

The first five minutes, the heartbeat, and the numbers that explain themselves

الدقائق الخمس الأولى لها قانون، ولا شيء يدخل أورا بصمت، والرقم الذي يتحرك يشرح نفسه، وكلمة المستخدم لا تضيع أبداً
17.1
Onboarding: progress always visible («٣ / ٧ · باقي دقيقتان») · the «لماذا نسأل؟» line under every question · the time promise honest and measured by Gate 4.
17.2
The Capture Receipt. Receipt within one second («وصل — جارٍ القراءة ✦») · disposition when processing completes — which signal it fed, or with dignity, «قُرئ — لم يلتقِ بإشارة بعد». Nothing enters Aura silently. Lineage runs both ways: fragments know their signals, signals their fragments.
17.3
Score Attribution. Every score movement carries its top cause, one line: «+٣ — إشارة دخلت مستوى Live أمس». The numeral stays Mono-Western (data); the cause's digits follow §4.1.
17.4
Session Safety. Every editing surface autosaves (≤10s) · re-auth restores exact state · leaving with unsaved changes warns once. «مسودتك محفوظة» must always be true. No user text is ever lost to auth, network, or navigation.
17.5
Four content states per surface: loading, empty, partial, error — all designed, each obeying §11. Gate 2 fails any surface missing one.

§18 · Exports, Email, Logo, Imagery & Charts

The most public pixels carry the strictest law

التقرير والشرائح وبطاقة المشاركة والبريد: أبعاد ثابتة، خطوط مدمجة، شعار له حرمه — ولا مصافحات مسرحية
18.1
The export kit: OG/share card, Report PDF, carousel, credential cards — fixed dimensions, embedded fonts, the headline pair ("Your expertise is invisible. Aura fixes that." / «حتى السوق يعرفك قبل ما يشوفك»), bronze ✦. The LinkedIn paste of aura-intel.org is a pitch, not a gray rectangle.
18.2
Email-safe stacks: Georgia for display, Tahoma/'Segoe UI' for Arabic (line-height 1.9 preserved), hex constants from §1, single column 600px, bulletproof buttons. Designed for the worst client, beautiful in the best.
18.3
The logo law: three variants (bronze · ink · night-bronze) · clear space = ✦ height on all sides · min 24px UI / 12mm print · never on signal orange, photos, stretched, or recolored.
18.4
Imagery: the only people pictured are real (the user). Decorative imagery is abstract, parchment-bronze duotone. Banned by name: stock handshakes, fake skylines, staged boardrooms. Avatars: circle, 1.5px bronze ring, Cormorant initial fallback (Cairo for Arabic names).
18.5
Chart grammar: the point of the chart wears bronze, everything else muted ink · hairline gridlines · Mono axes, horizontal labels, Western digits (charts are data) · time flows LTR on every surface · one viz per page · every chart ships its empty state.

Governance & Gates

§19 · The Four Gates

Quality is a checkpoint, not a hope — artifacts, never testimony

أربع بوابات: لكل أمر بنائي، ولكل شاشة، ولكل نشر، ولكل إثنين — ولا تُعبر بوابة بشهادة، بل بدليل ملموس
G1
Every prompt — Definition of Done: named snapshot · diagnose before fixing (code & DB quoted) · one concern, holistic across shared-data surfaces · tokens only · real tsc output pasted (exit 0 is a fact, "clean" is a claim) · SELECT-with-count before any DELETE, confirmed separately.
G2
Every surface — the Surface Pass: both modes (light canonical, night verified) · both directions (Arabic verified as Arabic) · contrast ≥4.5:1 text / ≥3:1 secondary · primitives only · all four content states (§17.5) · microcopy passes §11. Admin clause: back-office surfaces use the tokens and ramp, skip peaks and marks, pass a lighter G2 (contrast + tokens).
G3
Every publish — the Smoke Path: preview verified first → publish → Empty Cache & Hard Reload · golden path walked (Home → draft → edit → ship) · one Arabic + one English generation read aloud · one real paste into LinkedIn · prod spot-check.
G4
Every Monday — the Stopwatch: time-to-shipped <15 min, in Arabic and in English, switching mid-session · every hesitation written down · two users hit it → fix; three → P0 · week-two unprompted return is the "like" metric.
§20 · Governance & Non-Goals

How the law changes — and what Aura refuses to become

القانون يتغير باسم ما يمسه فقط: أي تعديل يسمّي المواد والنصوص والرموز التي يطالها — وإلا يسقط عند البوابة
20.1
The Precedence Law. Later supersedes earlier — but must say so by name: every amendment declares the sections it touches, the locked strings it rewrites (old and new side by side), and the tokens it adds through §1.2's grammar. A silent amendment fails the gate.
20.2
One living page. The current version at /admin/design-system is always the whole law; previous versions are history. One changelog. Print stylesheet included for partner-facing copies.
20.3
New surfaces are born inside the Coverage Map or they don't ship.
