# Plan: brand_pillars gets a real writer (v2)

One pipeline. No schema change (`brand_pillars text[]` already exists on `diagnostic_profiles`).

## 1. Shape of `brand_assessment_results`

Confirmed from the three most recent completed assessments. The current `brand-assessment` edge function returns a structured JSON block that `BrandAssessmentModal.handleSaveToIdentity` (lines 426–447) merges into `resultsObj` and persists. The relevant key is:

```jsonc
{
  "content_pillars": [
    "Clinical AI Workforce Impact in GCC Healthcare",
    "Patient Safety in AI-Enabled Clinical Pathways",
    "Building Executive Confidence in Healthcare AI Decisions"
  ],
  "primary_archetype": "...",
  "positioning_statement": "..."
}
```

These pillars are already user-specific — generated by the LLM from the user's answers, `sector`, and `auditScores` passed in at line 396. We just never write them to the column.

Legacy rows (one of the three empties) have only a prose `interpretation` with a `## YOUR 3 TOPICS` / `MY 3 AUTHORITY THEMES` section whose three bolded titles are the pillars.

## 2. Derivation rule (single source, mirrored)

A pure helper `derivePillars(results: Record<string, any>): string[]`:

1. If `Array.isArray(results.content_pillars)` → trim, drop empties, slice to 5. Return.
2. Else, parse `results.interpretation` prose: locate the `YOUR 3 TOPICS` / `MY 3 AUTHORITY THEMES` / `YOUR 3 AUTHORITY THEMES` heading and extract the first 3–5 bolded titles (`**Title**` or `* **Title:**`).
3. Else return `[]`.

Never invent generic themes; an empty array is the correct "I don't know" answer and the EF will handle it (section 5).

Files (identical implementations, separate runtimes):

- `src/lib/brandPillars.ts` — canonical.
- `supabase/functions/_shared/brandPillars.ts` — first line is the comment `// MUST mirror src/lib/brandPillars.ts — keep byte-identical` (same convention as `src/constants/seniority.ts`). Body is a byte-identical copy of the function.

## 3. Write at assessment completion

`src/components/BrandAssessmentModal.tsx` `handleSaveToIdentity` (lines 416–456): in the same `.update({...})` that already sets `brand_assessment_results` and `brand_assessment_completed_at`, also set:

```ts
brand_pillars: derivePillars(resultsObj),
```

Only writes pillars when derivation yields ≥1; otherwise omit the field so it isn't blanked.

## 4. Backfill for existing users

New edge function `supabase/functions/backfill-brand-pillars/index.ts`:

- **Auth: CRON_SECRET header only.** Require header `x-cron-secret` (or `Authorization: Bearer <CRON_SECRET>`) equal to `Deno.env.get("CRON_SECRET")`. Any mismatch or missing header → `401`. No fallback, no admin-role path, no preview-session bypass.
- Service-role client.
- `select user_id, brand_assessment_results from diagnostic_profiles where brand_assessment_completed_at is not null and (brand_pillars is null or cardinality(brand_pillars) = 0)`.
- For each row, run `derivePillars` (imported from `_shared/brandPillars.ts`). Update only when ≥1 pillar derived. Return `{ scanned, updated, skipped }`.

One-shot — invoked once by the developer via `curl` with `x-cron-secret`. Confirmed candidates today: 3 users with completed assessments and empty `brand_pillars`; 2 populate from `content_pillars`, 1 from prose parsing.

## 5. Remove the hardcoded fallback

`supabase/functions/generate-branded-post/index.ts` line 77 currently:

```ts
const brandPillars = (p.brand_pillars || []).join(", ") || "Strategy, Innovation, Leadership";
```

Change to a true empty-aware branch:

```ts
const pillarsArr = Array.isArray(p.brand_pillars) ? p.brand_pillars.filter(Boolean) : [];
const brandPillars = pillarsArr.join(", "); // "" when none
```

In the `systemPrompt` template, replace the unconditional `BRAND PILLARS: ${brandPillars}` line with a conditional block — when `pillarsArr.length === 0`, the entire `BRAND PILLARS: ...` line is omitted from the prompt rather than emitting `BRAND PILLARS: ` or a fabricated triplet.

No other file in the codebase contains the `"Strategy, Innovation, Leadership"` literal (verified by grep before this plan).

## Out of scope

- Other readers of `brand_pillars` (~20 sites) — they already tolerate empty arrays.
- The legacy `ExecutiveDiagnostic.tsx:280` writer — left untouched.
- Schema changes — none.
